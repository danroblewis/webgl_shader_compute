Custom WebGL Rendering LAYER 1: GPU Access

This example demonstrates direct WebGL access (Layer 1). The simulation texture is rendered directly to the screen using a custom fragment shader.

⚡ Zero CPU involvement in rendering - pure GPU→GPU pipeline!

Layer 1 API: WebGL Texture Access

// Get raw WebGL resources
const texture = sim.getTexture();         // WebGLTexture
const gl = sim.getContext();             // WebGLRenderingContext
const compute = sim.getComputeEngine();  // For custom kernels

// Bind simulation texture directly
gl.activeTexture(gl.TEXTURE0);
gl.bindTexture(gl.TEXTURE_2D, texture);

// Custom fragment shader reads simulation state
const renderShader = `
    uniform sampler2D u_simTexture;
    
    void main() {
        float alive = texture2D(u_simTexture, v_texCoord).r;
        vec3 color = alive > 0.5 
            ? vec3(0.0, 1.0, 0.5)  // Alive: cyan
            : vec3(0.0, 0.0, 0.1); // Dead: dark blue
        gl_FragColor = vec4(color, 1.0);
    }
`;

// Render directly: GPU→GPU, no CPU download!
gl.drawArrays(gl.TRIANGLES, 0, 6);
